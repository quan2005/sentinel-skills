# Agent Skills Specification

本文档定义了Agent技能的标准规范和最佳实践。

## 技能目录结构

每个技能应该放在独立的目录中：

```
skills/
└── skill-name/
    ├── SKILL.md          # 技能说明文档 (必需)
    └── [其他文件...]     # 技能相关文件 (可选)
```

## SKILL.md 规范

### 基本结构

```markdown
# 技能名称

简短的技能描述（一句话概括）。

## 何时使用

明确描述触发该技能的场景、条件或用户意图。

## 功能描述

详细说明技能的能力和用途。

## 使用示例

提供1-3个具体的使用示例。

## 技术实现

(可选) 简要说明技术实现方式。

## 注意事项

(可选) 使用限制、依赖条件等。
```

### 命名规范

- **技能目录**: 小写字母、连字符分隔，如 `data-analyzer`
- **技能名称**: 简洁明了，体现核心功能
- **文档语言**: 优先使用中文，技术术语可用英文

### 内容要求

1. **描述准确**: 避免夸大或模糊的描述
2. **示例具体**: 提供真实可用的使用场景
3. **边界清晰**: 明确技能的适用范围和限制

## 技能分类

建议按功能领域组织技能：

- **数据处理**: 数据分析、转换、可视化
- **文档处理**: 文档生成、格式转换、内容提取
- **开发工具**: 代码生成、调试、测试
- **创意工具**: 内容创作、设计辅助
- **自动化**: 流程自动化、任务管理
- **分析工具**: 市场分析、竞品分析
- **学习辅助**: 知识整理、学习计划

## 开发指南

### 创建新技能

1. 在 `skills/` 目录下创建新文件夹
2. 编写 `SKILL.md` 文档
3. 添加必要的支持文件
4. 测试技能效果

### 技能设计原则

- **单一职责**: 每个技能专注于一个核心功能
- **用户导向**: 从用户需求出发设计功能
- **易用性**: 提供清晰的说明和示例
- **可靠性**: 确保技能稳定可用

### 质量标准

- 文档完整，符合规范
- 功能描述准确无歧义
- 示例真实可用
- 代码结构清晰（如有代码）

## 维护和更新

- 定期检查技能的有效性
- 更新过时的信息和示例
- 根据用户反馈改进技能
- 保持与Claude Code版本的兼容性
