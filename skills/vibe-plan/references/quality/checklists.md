# Quality Checklists for Constitution Documents

按场景分类的宪法文档质量检查清单。

---

## 通用宪法检查清单（所有场景）

### 目标质量
- [ ] 目标清晰：一句话能概括要达成什么
- [ ] 目标具体：避免模糊词汇（如"优化"、"改进"）
- [ ] 目标可衡量：有明确的成功标准

### 条款质量
- [ ] 条款原子性：每个条款描述一个独立方面
- [ ] 抽象原则：条款不规定具体执行步骤
- [ ] 可验证性：每个条款都有明确的验证指导
- [ ] 条款完整：覆盖核心需求

### 边界管理
- [ ] 有"不做的事项"防止蔓延
- [ ] 明确的排除范围
- [ ] 每个排除项都有原因说明

### 可执行性
- [ ] 另一位 AI 能理解并达成目标
- [ ] 验证方式可操作
- [ ] 规模适当（预估 ≤ 2 小时可完成）

---

## 开发任务宪法检查清单

### 需求完整性
- [ ] 功能范围明确
- [ ] 技术约束说明
- [ ] 集成点识别
- [ ] 依赖关系澄清

### 质量标准
- [ ] 代码质量条款（如：遵循现有模式）
- [ ] 测试验证方式
- [ ] 性能要求（如适用）
- [ ] 兼容性要求

### 集成考虑
- [ ] 不破坏现有功能
- [ ] 遵循项目约定
- [ ] API 变更说明（如适用）
- [ ] 数据库变更说明（如适用）

---

## 内容创作宪法检查清单

### 目标定位
- [ ] 目标读者明确（具体人群/水平）
- [ ] 内容类型清晰（教程/观点/分析）
- [ ] 篇幅范围说明

### 内容质量
- [ ] 结构要求明确（几部分/什么章节）
- [ ] 原创性标准
- [ ] 准确性验证方式
- [ ] 可读性要求

### 格式输出
- [ ] 最终格式说明（Markdown/PDF/视频等）
- [ ] 发布平台要求（如适用）
- [ ] 配套素材要求（图片/代码示例等）

---

## 学习计划宪法检查清单

### 起点评估
- [ ] 当前基础水平说明
- [ ] 前置知识要求
- [ ] 学习资源类型偏好

### 目标定义
- [ ] 学习目标具体化（能看懂/能写/深入理解）
- [ ] 可演示的成果
- [ ] 时间投入规划

### 里程碑设计
- [ ] 关键节点可衡量
- [ ] 验证方式明确
- [ ] 调整机制说明

---

## 方案设计宪法检查清单

### 范围明确
- [ ] 目标规模量化（人数/时长/预算）
- [ ] 参与方识别
- [ ] 约束条件完整

### 可行性
- [ ] 资源需求评估
- [ ] 时间规划合理
- [ ] 风险识别与应对

### 成功标准
- [ ] 可量化的成功指标
- [ ] 验收方式明确
- [ ] 备选方案说明

---

## 快速验证（5秒检查）

```
[ ] 目标：一句话能说清
[ ] 条款：每个都能验证
[ ] 排除：有"不做的事项"
[ ] 可执行：给别的 AI 也能懂
```
