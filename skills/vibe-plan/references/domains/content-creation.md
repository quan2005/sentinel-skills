# Content Creation Constitution Patterns

内容创作相关宪法的条款模板和验证方式。

当宪法涉及文章、视频、演讲稿、任何内容产出时使用。

---

## 条款模板

### 目标读者条款

```
## 条款：目标读者适配

内容针对具体读者群体，使用适当的语言深度和风格。

**验证方式**：
- [ ] 目标读者明确定义（如：中级前端开发者）
- [ ] 术语适合读者水平（或已解释）
- [ ] 例子与读者相关
- [ ] 预备知识要求说明
```

### 结构清晰条款

```
## 条款：内容结构

内容有清晰的结构，易于导航和阅读。

**验证方式**：
- [ ] 有明确的章节划分
- [ ] 有目录或导航
- [ ] 段落间有逻辑过渡
- [ ] 重要概念有强调
```

### 代码可运行条款

```
## 条款：代码可运行

文中的代码示例可以直接运行并产生预期结果。

**验证方式**：
- [ ] 每个代码块已测试运行
- [ ] 包含必要的 import/setup
- [ ] 输出结果已验证
- [ ] 环境要求已说明
```

### 原创性条款

```
## 条款：内容原创

内容是原创或有明确引用来源。

**验证方式**：
- [ ] 核心观点为原创或综合
- [ ] 引用内容有明确出处
- [ ] 非抄袭他人作品
- [ ] 使用工具检测（如适用）
```

### 可读性条款

```
## 条款：可读性

内容使用流畅的语言，避免翻译腔和冗余表达。

**验证方式**：
- [ ] 通读全文，语句通顺自然
- [ ] 无明显翻译痕迹
- [ ] 专业术语有解释或链接
- [ ] 句子长度适中（平均 < 20 字）
```

### 准确性条款

```
## 条款：内容准确

内容中的事实、数据、技术细节准确无误。

**验证方式**：
- [ ] 技术声明已验证
- [ ] 数据有来源或测试
- [ ] 引用原文已核对
- [ ] 代码示例已测试
```

---

## 内容类型特定条款

### 教程类内容
```
## 条款：学习路径

教程提供清晰的学习路径和循序渐进的讲解。

**验证方式**：
- [ ] 有先决条件说明
- [ ] 步骤按逻辑顺序排列
- [ ] 每步有预期结果说明
- [ ] 有故障排除指南
```

### 观点类内容
```
## 条款：论点清晰

观点类内容有明确的论点和充分的论证。

**验证方式**：
- [ ] 核心观点在开头明确陈述
- [ ] 每个论点有论据支持
- [ ] 考虑了反驳观点
- [ ] 结论与论点一致
```

### 实战类内容
```
## 条款：实战价值

实战类内容提供可立即应用的知识或工具。

**验证方式**：
- [ ] 包含真实场景案例
- [ ] 解决方案可直接复用
- [ ] 提供完整代码/资源
- [ ] 有实际效果展示
```

---

## 输出格式条款

### Markdown 输出
```
## 条款：Markdown 规范

Markdown 格式遵循最佳实践，渲染一致。

**验证方式**：
- [ ] 标题层级正确（无跳跃）
- [ ] 代码块指定语言
- [ ] 链接可点击
- [ ] 图片有 alt 文本
```

### 视频输出
```
## 条款：视频质量

视频内容具有良好的视听质量。

**验证方式**：
- [ ] 音频清晰无杂音
- [ ] 画面稳定清晰
- [ ] 字幕/文字可读
- [ ] 时长合理（< 30 分钟）
```

### 演讲稿输出
```
## 条款：演讲可讲性

演讲稿适合口头表达，易于听众理解。

**验证方式**：
- [ ] 句子简短有力
- [ ] 避免复杂从句
- [ ] 有停顿标记（如适用）
- [ ] 预估时长合理
```

---

## 内容检查清单

```
发布前验证：
- [ ] 目标读者明确
- [ ] 结构清晰完整
- [ ] 代码示例可运行
- [ ] 引用有出处
- [ ] 无抄袭内容
- [ ] 事实数据准确
- [ ] 语言流畅自然
- [ ] 格式渲染正确
- [ ] 适合目标平台
```
