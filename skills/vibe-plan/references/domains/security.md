# Security Constitution Patterns

安全相关宪法的条款模板和验证方式。

当宪法涉及用户数据、认证、支付、权限等敏感操作时使用。

---

## 核心原则

```
安全思维假设：
- 所有输入都是恶意的
- 所有用户都会尝试突破限制
- 密钥终会被暴露（限制泄露范围）
- 依赖包可能存在漏洞
- 系统终会被攻破（减少损失）
```

---

## 条款模板

### 输入验证条款

```
## 条款：输入验证

所有外部输入都经过验证，拒绝不符合预期的数据。

**验证方式**：
- [ ] 检查类型、格式、长度、范围
- [ ] 使用白名单而非黑名单
- [ ] 拒绝未知字段
- [ ] 按输出上下文清理（HTML、SQL、shell）
```

### 认证授权条款

```
## 条款：认证与授权

每个受保护请求都验证用户身份和权限。

**验证方式**：
- [ ] 用户已通过身份认证
- [ ] 用户有权访问该资源
- [ ] 用户有权执行该操作
- [ ] 检查资源所有权（不仅检查角色）
```

### 密钥管理条款

```
## 条款：密钥管理

无硬编码密钥，无密钥泄露到日志或错误信息。

**验证方式**：
- [ ] 代码中无硬编码密钥
- [ ] 使用环境变量或密钥管理服务
- [ ] 日志中无敏感信息
- [ ] 错误消息不暴露内部细节
- [ ] 不同环境使用不同密钥
```

### 密码安全条款

```
## 条款：密码安全

用户密码使用安全的哈希算法存储，永不明文存储。

**验证方式**：
- [ ] 使用 bcrypt 或 argon2 哈希（NOT MD5/SHA1）
- [ ] salt 自动处理
- [ ] 日志中无密码
- [ ] 错误消息不透露密码是否存在
```

### 数据传输条款

```
## 条款：安全传输

所有敏感数据通过 HTTPS 传输，无明文通信。

**验证方式**：
- [ ] 所有端点强制 HTTPS
- [ ] TLS 1.2+ 仅允许
- [ ] 证书有效
- [ ] Cookie 设置 Secure 标志
```

### 数据存储条款

```
## 条款：数据保护

敏感数据在存储时加密，备份同样加密。

**验证方式**：
- [ ] 敏感字段加密存储
- [ ] 备份文件加密
- [ ] 密钥安全管理
- [ ] 访问日志记录
```

### SQL 注入防护条款

```
## 条款：SQL 安全

数据库查询使用参数化查询，防止 SQL 注入。

**验证方式**：
- [ ] 所有查询使用参数化（? 或 $1）
- [ ] 无字符串拼接构建 SQL
- [ ] ORM 使用正确（无 raw query 拼接）
```

### XSS 防护条款

```
## 条款：XSS 防护

用户生成内容在渲染前经过清理。

**验证方式**：
- [ ] 默认使用 textContent 而非 innerHTML
- [ ] 需要 HTML 时使用 DOMPurify 清理
- [ ] 框架自动转义（React/Vue）
- [ ] CSP 头设置
```

---

## 安全检查清单

在发布涉及安全的宪法相关成果前：

```
- [ ] 无硬编码密钥
- [ ] 所有用户输入已验证
- [ ] SQL 使用参数化查询
- [ ] HTML 输出已转义
- [ ] 受保护路由有认证检查
- [ ] 授权检查资源所有权
- [ ] 密码正确哈希（bcrypt/argon2）
- [ ] HTTPS 强制
- [ ] 安全响应头已设置
- [ ] 依赖包已审计
- [ ] 敏感数据已加密
- [ ] 日志不包含密钥
- [ ] 错误消息不暴露内部信息
- [ ] 敏感端点有速率限制
```

---

## 安全条款 Red Flags

```
❌ "用户输入是安全的" — 如何验证？
❌ "使用基本认证" — 太模糊
❌ "防止黑客攻击" — 不具体

✅ "所有外部输入验证类型、格式、长度"
✅ "使用 bcrypt 加密密码，work factor 12"
✅ "API 端点速率限制：每分钟 5 次登录尝试"
```
