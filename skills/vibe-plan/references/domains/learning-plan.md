# Learning Plan Constitution Patterns

学习计划相关宪法的条款模板和验证方式。

当宪法涉及技能学习、阅读计划、研究课题时使用。

---

## 核心原则

```
有效学习的特点：
- 目标具体可衡量
- 起点清晰评估
- 路径循序渐进
- 成果可演示
- 反馈及时有效
```

---

## 条款模板

### 起点评估条款

```
## 条款：起点明确

学习计划基于当前水平制定，目标可达成。

**验证方式**：
- [ ] 当前技能水平明确描述
- [ ] 前置知识要求说明
- [ ] 与目标的差距评估
- [ ] 时间投入现实可行
```

### 目标可衡量条款

```
## 条款：目标可衡量

学习目标具体，可判断是否达成。

**验证方式**：
- [ ] 目标可演示（如：能写出 X）
- [ ] 有明确验收标准
- [ ] 不是模糊的"理解"或"掌握"
- [ ] 可被他人评估
```

### 里程碑条款

```
## 条款：渐进里程碑

学习路径分解为可达成的小里程碑。

**验证方式**：
- [ ] 里程碑按难度递增
- [ ] 每个里程碑有明确产出
- [ ] 里程碑间有逻辑依赖
- [ ] 每个里程碑可验证
```

### 实践项目条款

```
## 条款：实践驱动

学习通过实际项目巩固，而非仅阅读/观看。

**验证方式**：
- [ ] 每个概念有对应练习
- [ ] 有完整项目作为最终成果
- [ ] 项目难度与学习进度匹配
- [ ] 项目可展示给他人
```

### 资源质量条款

```
## 条款：优质资源

学习资源来自可靠来源，内容质量高。

**验证方式**：
- [ ] 资源有明确来源/作者
- [ ] 内容相对较新（技术类）
- [ ] 有良好评价或推荐
- [ ] 与学习目标直接相关
```

### 反馈机制条款

```
## 条款：学习反馈

有方式验证学习效果，获得反馈。

**验证方式**：
- [ ] 有自测题目或练习
- [ ] 有项目产出可评估
- [ ] 有他人反馈渠道（如适用）
- [ ] 有调整学习路径的机制
```

---

## 学习类型特定条款

### 编程语言学习
```
## 条款：语法掌握

掌握语言的核心语法和惯用法。

**验证方式**：
- [ ] 能阅读该语言代码
- [ ] 能独立编写中等复杂度程序
- [ ] 理解语言特有概念（如 Rust 所有权）
- [ ] 能调试常见错误

## 条款：生态熟悉

熟悉该语言的常用库和工具。

**验证方式**：
- [ ] 了解包管理器使用
- [ ] 知道常用库
- [ ] 能搭建开发环境
- [ ] 了解调试工具
```

### 框架/库学习
```
## 条款：核心概念理解

理解框架的核心概念和设计模式。

**验证方式**：
- [ ] 能解释框架的核心概念
- [ ] 理解框架的目录结构约定
- [ ] 知道组件/模块的组织方式
- [ ] 理解数据流动

## 条款：实战应用

能用框架完成典型任务。

**验证方式**：
- [ ] 能创建新项目
- [ ] 能实现 CRUD 功能
- [ ] 能处理路由/状态
- [ ] 能部署应用
```

### 理论/概念学习
```
## 条款：概念理解

能用自己的话解释核心概念。

**验证方式**：
- [ ] 能给出概念的简单定义
- [ ] 能举出具体例子
- [ ] 能说明概念间的关联
- [ ] 能解释概念的应用场景

## 条款：知识体系

将新知识整合到已有知识体系中。

**验证方式**：
- [ ] 能画出概念关系图
- [ ] 能说明与已知概念的联系
- [ ] 能区分易混概念
- [ ] 能在不同场景间迁移知识
```

### 阅读/研究计划
```
## 条款：阅读输出

阅读有明确的输出成果。

**验证方式**：
- [ ] 每章/节有笔记
- [ ] 有总结或思维导图
- [ ] 有自己的思考和问题
- [ ] 能向他人讲解内容

## 条款：进度追踪

阅读进度可追踪，保持持续性。

**验证方式**：
- [ ] 有明确的阅读计划（章节/时间）
- [ ] 有进度记录方式
- [ ] 有应对中断的策略
- [ ] 有完成标准
```

---

## 学习成果验证条款

### 可演示成果
```
## 条款：成果可演示

学习结束有可展示的具体成果。

**验证方式**：
- [ ] 有完整项目/作品
- [ ] 可分享给他人查看
- [ ] 能现场演示
- [ ] 能解释实现思路
```

### 知识保留
```
## 条款：知识保留

学到的知识能在一段时间后仍能回忆。

**验证方式**：
- [ ] 一周后能回忆核心概念
- [ ] 不依赖资料能完成练习
- [ ] 能教给他人
- [ ] 能应用于新场景
```

## 学习计划检查清单

```
制定学习计划时：
- [ ] 目标具体可衡量
- [ ] 起点水平明确
- [ ] 里程碑可达成
- [ ] 有实践项目
- [ ] 资源质量可靠
- [ ] 时间规划现实
- [ ] 有反馈机制
- [ ] 有完成标准

开始学习时：
- [ ] 开发环境已搭建
- [ ] 资源已收集
- [ ] 日程已安排
- [ ] 进度记录方式确定
```

---

## 学习计划 Red Flags

```
❌ "精通 XXX" — 太宽泛，无法验证
❌ "学习 XXX" — 无具体目标
❌ "一个月成为专家" — 不现实

✅ "能独立使用 React 创建 Todo 应用"
✅ "能解释并实现快速排序算法"
✅ "能阅读中等复杂度的 Python 代码"
```
