# ARCH.md 固定章节结构

## 铁律

1. 章节编号与标题逐字固定：不新增、不删除、不重排、不改名
2. 章节内部格式自由
3. 增量更新定位到具体章节，本次变更不涉及的章节一个字不动

## 变更定位表

一个变更可能落多章：

| 章节 | 收录内容 | 典型触发变更 |
|---|---|---|
| 1. 技术栈 | 语言、框架、关键依赖及版本 | 新依赖、框架升级、运行时变更 |
| 2. 系统分层 | 层次划分与职责边界 | 新增层、层职责调整、模块归属变化 |
| 3. 核心设计决策 | 选型的背景 / 选项 / 选择 / 代价 | 技术选型、架构取舍、决策被推翻 |
| 4. 前端架构 | 组件体系、状态管理、路由 | 组件库调整、状态方案变更 |
| 5. 后端架构 | 服务划分、接口、存储 | 新服务、接口契约变更、存储选型 |
| 6. 数据流 | 关键数据路径、事件流 | 新数据路径、事件链路变化 |
| 7. 横切关注点 | 鉴权、日志、监控、错误处理、i18n | 中间件、全局策略变更 |
| 8. 技术债与已知限制 | 已知妥协、待还的债 | 新引入的妥协、债务偿还 |

## 更新纪律

- 第 3 章是决策史：被推翻的决策标注「已被 X 取代（{日期}）」，不删除
- 第 8 章是唯一允许删条目的章节：债还清即删，并在 commit message 说明
- 初始化时以读代码归纳为主、story/design 为辅；不确定处留 `<!-- TODO -->`，不编造
