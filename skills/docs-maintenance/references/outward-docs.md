# 对外文档维护规则（README / llms.txt / 使用说明 / 开发说明）

读者是项目的使用者与贡献者——**可能是人，也可能是 Agent**。"按需"修饰的是更新而非存在性：

- 触发判据不命中 → 不更新（这是正常出口）
- 触发判据命中、文档已存在 → 增量更新
- 触发判据命中、文档不存在 → **直接初始化，无需用户发起**

## 分工与边界

| 文档 | 读者 | 职责 |
|---|---|---|
| README.md | 所有人（人类浏览入口） | 门面：是什么、快速开始、文档导航。保持简短，详情链接 docs |
| llms.txt | Agent（机读导航入口） | 规范化索引：摘要 + 分节链接列表，供 Agent 按需取用文档 |
| 用户使用说明 | 使用者（人或 Agent） | 怎么用 |
| 技术开发说明 | 贡献者（人或 Agent） | 怎么改 |

README 与 llms.txt 同源同步：同一套文档结构，一个给人浏览，一个给 Agent 机读。

## 文件识别与初始化位置

按仓库现状识别既有文档；识别不到且需初始化时用默认路径：

| 类型 | 既有文档常见位置 | 初始化默认路径 |
|---|---|---|
| README | 仓库根 README.md | `README.md` |
| llms.txt | 仓库根 llms.txt | `llms.txt` |
| 用户使用说明 | README 的使用章节、USAGE.md、docs/ 下用户文档 | `docs/user-guide.md` |
| 技术开发说明 | DEVELOPMENT.md、CONTRIBUTING.md、docs/ 下开发文档 | `docs/dev-guide.md` |

仓库已有 docs 目录惯例时随其惯例。

## 触发判据

| 类型 | 更新触发（任一命中） |
|---|---|
| README | 项目定位 / 核心能力清单变化；快速开始路径变化（安装命令、最小运行示例）；docs 下文档增删移需登记导航 |
| llms.txt | 与 README 导航完全同触发：docs 文档增删移、文档定位（一句话说明）变化、项目摘要变化 |
| 用户使用说明 | 新 AC 落地为用户可感知行为；命令 / 参数 / 配置项 / 默认值变化；升级或迁移注意事项 |
| 技术开发说明 | 构建 / 测试 / 调试 / 发布步骤变化；环境与依赖要求变化；目录结构或模块入口变化 |

story 的 AC 是用户使用说明的天然索引——AC 描述的本就是用户可观察行为。

## llms.txt 格式铁律（llmstxt.org 规范）

节序固定，可被解析器机读：

1. H1 项目名（唯一必需节）
2. blockquote 一段摘要：理解全文所需的关键信息
3. 零或多段不含标题的说明（如何解读下列文件、重要注意事项）
4. 零或多个 H2 文件列表节，条目格式 `- [名称](路径): 一句话说明`
5. `## Optional` 节有特殊语义：上下文紧张时可跳过的次要链接，只放可舍弃的内容

行文：简短清晰、每个链接必带说明、不留未解释的术语。

## 初始化纪律

- 最小可用：先完整覆盖本次变更，存量内容按读代码与已 verified 的历史 story/design 归纳
- 不确定处留 `<!-- TODO -->`，不编造（llms.txt 内用普通文字注明待补，保持可解析）
- 结构以读者任务为纲：README（一句话定位 / 核心能力 / 快速开始 / 文档导航）；llms.txt（按上节格式铁律）；用户文档（安装 / 使用 / 配置）；开发文档（环境 / 构建 / 测试 / 发布）。按项目实际取舍

## 双读者写作铁律

- README 是门面不是手册：任何超出快速开始深度的内容，链接到 docs 对应文档，不展开
- 用户文档只写"怎么用"：零实现细节、零内部术语
- 开发文档只写"怎么改"：不复制 ARCH.md / AGENTS.md 内容，需要背景时链接对应章节——单一事实源，防分叉
- 为 Agent 读者负责：标题锚点保持稳定（Agent 依赖锚点定位）、命令可直接复制执行、措辞简短无歧义、不写营销话术
- 示例命令与输出也是内容：行为变了示例必须跟着变，过时示例 = 不一致

## 更新纪律

- 增量定位到受影响的既有章节，尊重文档现有结构（ARCH.md 之外的文档不强制章节）
- 本次变更不涉及的章节一个字不动
- README 导航与 llms.txt 链接列表必须同次更新，不允许只改其一
