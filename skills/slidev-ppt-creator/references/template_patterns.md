# Template Patterns Reference

## Business Presentation Patterns

### Product Launch Template
**Structure:**
1. Title slide - Product name and tagline
2. Market opportunity - Problem and market size
3. Solution overview - Product introduction
4. Key features - Core capabilities
5. Technical advantages - Innovation points
6. Business model - Revenue streams
7. Go-to-market strategy - Launch plan
8. Team introduction - Key people
9. Investment opportunity - Funding needs
10. Vision and roadmap - Future plans

**Content Patterns:**
- Focus on market validation and business metrics
- Include competitor analysis
- Emphasize ROI and business value
- Use professional, corporate styling

### Company Introduction Template
**Structure:**
1. Cover slide - Company logo and name
2. Company overview - Mission and vision
3. History and milestones - Key achievements
4. Products and services - Core offerings
5. Market position - Industry standing
6. Customer success stories - Case studies
7. Team and culture - People and values
8. Financial highlights - Performance metrics
9. Future outlook - Growth plans
10. Contact information - Next steps

**Content Patterns:**
- Balance between past achievements and future vision
- Include customer testimonials
- Show financial growth trends
- Emphasize company values and culture

### Investment Pitch Template
**Structure:**
1. Problem statement - Market pain point
2. Solution overview - Product/service
3. Market size and opportunity - TAM/SAM/SOM
4. Business model - How we make money
5. Traction and milestones - Progress to date
6. Competitive landscape - Market positioning
7. Go-to-market strategy - Growth plan
8. Team overview - Key players
9. Financial projections - 3-5 year forecast
10. Funding needs - Use of funds

**Content Patterns:**
- Focus on scalability and market opportunity
- Include financial projections and key metrics
- Emphasize team expertise and track record
- Use data-driven arguments throughout

## Technical Presentation Patterns

### Technical Talk Template
**Structure:**
1. Title slide - Topic and speaker
2. Introduction - Background and context
3. Problem statement - Technical challenge
4. Current solutions and limitations
5. Proposed solution - Architecture/design
6. Implementation details - Code examples
7. Performance analysis - Benchmarks/results
8. Lessons learned - Challenges and insights
9. Future improvements - Next steps
10. Q&A - Discussion

**Content Patterns:**
- Include code snippets and architecture diagrams
- Focus on technical implementation details
- Provide performance metrics and comparisons
- Share lessons learned and best practices

### Architecture Review Template
**Structure:**
1. Overview - Current system state
2. Business requirements - What needs to change
3. Architecture principles - Design guidelines
4. Proposed architecture - New system design
5. Component breakdown - Key modules
6. Data flow - Information movement
7. Technology stack - Tools and frameworks
8. Migration strategy - Transition plan
9. Risk assessment - Potential issues
10. Timeline and resources - Implementation plan

**Content Patterns:**
- Use detailed architecture diagrams
- Include technology justification
- Provide migration roadmap
- Address scalability and maintainability

### Tutorial/Workshop Template
**Structure:**
1. Introduction - Learning objectives
2. Prerequisites - Setup requirements
3. Concept overview - Theory background
4. Step-by-step guide - Hands-on exercises
5. Code walkthrough - Detailed explanations
6. Common pitfalls - Issues to avoid
7. Best practices - Industry standards
8. Advanced topics - Next steps
9. Resources - Further learning
10. Summary - Key takeaways

**Content Patterns:**
- Include interactive coding exercises
- Provide step-by-step instructions
- Use code examples with explanations
- Include troubleshooting section

## Educational Presentation Patterns

**Structure:**
1. Title slide - Topic and session
2. Learning objectives - What students will learn
3. Previous session review - Context connection
4. Main concepts - Key theories/frameworks
5. Examples and illustrations - Real-world applications
6. Interactive exercise - Student engagement
7. Summary of key points - Reinforcement
8. Next class preview - Preparation
9. Resources and materials - Further study
10. Q&A - Student questions

**Content Patterns:**
- Clear learning objectives for each section
- Include formative assessments
- Use real-world examples and analogies
- Provide additional learning resources

2. Why this matters - Business relevance
3. Current state assessment - Skill baseline
4. New concepts/frameworks - Core content
5. Practical application - Hands-on practice
6. Skill check - Understanding verification
7. Implementation plan - Applying at work
8. Resources and tools - Ongoing support
9. Action items - Next steps
10. Follow-up plan - Continued learning

**Content Patterns:**
- Focus on practical application
- Include skill assessments
- Provide implementation guidance
- Emphasize on-the-job application

## Visual Element Patterns

### Chart Usage Guidelines
- **Bar charts**: Comparisons, rankings, time series
- **Pie charts**: Proportions, compositions, percentages
- **Line charts**: Trends over time, continuous data
- **Area charts**: Cumulative totals, volume over time
- **Scatter plots**: Correlations, distributions

### Diagram Usage Guidelines
- **Flowcharts**: Processes, decision flows, workflows
- **Architecture diagrams**: Systems, software, infrastructure
- **Timeline diagrams**: Project schedules, historical events
- **Hierarchy diagrams**: Organizational structures, classifications
- **Network diagrams**: Relationships, connections, systems

### Color Scheme Patterns
- **Business**: Blue (trust), gray (professional), orange (energy)
- **Technical**: Purple (innovation), green (growth), red (importance)
- **Education**: Purple (knowledge), blue (calm), orange (creativity)
- **General**: Green (positive), blue (stable), gray (neutral)

## Slide Type Patterns

### Title Slides
- Large, readable title
- Subtitle for context
- Author/presenter information
- Date or event identifier
- Clean, uncluttered design

### Content Slides
- Clear heading hierarchy
- One main idea per slide
- Supporting visuals or data
- Minimal text (6x6 rule: 6 words x 6 lines)
- Consistent formatting

### Transition Slides
- Section dividers
- Chapter introductions
- Summary points
- Visual consistency
- Progress indicators

### Closing Slides
- Key takeaways summary
- Contact information
- Call to action
- Thank you message
- Q&A invitation
