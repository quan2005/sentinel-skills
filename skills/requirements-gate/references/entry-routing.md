# 入口路由

PM 输入处于哪种状态，决定从流程哪一步进。判断顺序：先看有没有 KPI，再看是不是已有草稿。

```
PM 输入
  ├─ 含 KPI / 数字目标？ ──是──→ BRIDGE ──→ 脊柱五问
  │         │否
  ├─ 是已有 PRD 草稿？ ──是──→ 草稿体检 ──→ 脊柱（缺哪问追哪问）
  │         │否
  └─────────────────────────→ 脊柱 Q1
```

## 四类入口

| 入口 | 识别信号 | 第一动作 |
|---|---|---|
| **业务目标** | 含 KPI/OKR/数字目标："次留 +5%""提升转化" | 进 BRIDGE，要 ≥2 假设 |
| **草稿体检** | PM 已有一份 PRD/文档，要"体检""看看行不行" | 对照脊柱五问扫描，列缺口报告，缺哪问追哪问 |
| **模糊想法** | 短句、转述、一个方案名："老板说做个 AI 助手" | 直入脊柱 Q1，按原型 A 拦截 |
| **用户反馈** | 零散反馈、吐槽、工单摘录 | 直入脊柱 Q1，按原型 B 追到具体角色场景 |

## 草稿体检的扫描方式

拿到草稿，逐条对照脊柱五问，生成扫描报告：

| 脊柱项 | 草稿是否覆盖 | 缺口 |
|---|---|---|
| Q1 谁+做什么 | ✅/❌ | {缺什么} |
| Q2 现状 | ✅/❌ | |
| Q3 失败模式 | ✅/❌ | |
| Q4 成功度量 | ✅/❌ | |
| Q5 三类边界 | ✅/❌ | |

报告给 PM 看后，只追缺的项，不重复已写清的——草稿体检的价值是精准补缺，不是从头再问一遍。
