# 安装说明

> 本文件面向人类读者，不属于技能上下文（Claude 触发技能时只加载 SKILL.md 及其引用的 references/assets）。

## 设计原则

- **需求识别与分级 100% 由 prompt 工程完成**：判断"这是不是开发需求、该走 L0–L3 哪一级"的逻辑全部在 SKILL.md 内，由模型执行。hook 不含任何判断代码。
- **hook 只做一件事**：在每条用户消息前静态注入一行项目约定。相比写在 CLAUDE.md，逐条注入能让规则在长会话中始终贴近最新消息，不被稀释。
- **静态 echo，零延迟**：官方文档指出 UserPromptSubmit hook 会阻塞模型处理直至完成（默认超时 30 秒），任何脚本或 LLM 评估都在给每条消息加延迟，静态输出没有这个代价。
- **注入文本是陈述句**：官方文档提示，命令式的越权指令文本可能触发 Claude 的提示注入防御（被当作可疑内容上报而非遵循），因此注入内容写成"本项目约定：……"的事实陈述。

## 安装步骤

1. 技能目录放到 `~/.claude/skills/requirements-gate/`（个人级）或项目内 `.claude/skills/requirements-gate/`。
2. 将 `hooks/settings.example.json` 中的 `hooks` 段合并进 settings.json：
   - 项目级：`.claude/settings.json`
   - 个人级：`~/.claude/settings.json`
   - echo 命令不含任何文件路径，两级配置通用，无需改动。
3. 验证：发一条含"新增 / 实现"的消息，Claude 应主动进入门禁流程；发一条纯提问，应正常回答不受打扰。

## 跳过约定

单条消息带 `[skip-gate]` 或 `[跳过门禁]` 即跳过门禁。该约定由注入文本与 SKILL.md 的 L0 规则共同保障——同样是纯 prompt 实现，没有脚本参与。

## 备选与升级路径

- **最简方案（不装 hook）**：把注入文本那行约定直接写进 CLAUDE.md。官方文档对"不变的指令"推荐此方式；代价是长会话中规则可能被压缩稀释，门禁纪律随之松动。两者可并存。
- **硬门禁（按需升级）**：若试用后发现 Claude 仍会"偷跑写代码"，可增加 PreToolUse hook，在 Edit/Write 工具调用前检查当前任务是否存在 approved 的 story.md，不存在则阻断。这是唯一具备强制阻断力的位置，建议软门禁跑顺后再做。

## 上下文成本

每条消息注入约 70 字，且会留在会话历史中累积，因此注入文本刻意压成一行。如需进一步降噪，可改用 CLAUDE.md 方案。
